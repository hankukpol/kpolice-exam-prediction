## 핵심 결론 (한 줄)

> **우리 사이트는 분석 엔진은 이미 강하다. 소단기처럼 "시험 시즌 운영 허브"로 보이게 만드는 것이 지금 해야 할 일이다.**

두 AI 분석 모두 같은 결론에 도달했다.
소단기의 강점은 **계산식이나 기능 수가 아니라 운영 리듬과 사용자 동선 설계**다.
우리 사이트는 데이터 품질과 분석 깊이에서 오히려 앞서 있다.

소단기와 비교해 우리가 이미 더 나은 부분. **절대 훼손하지 말 것.**

| 강점 | 구현 파일 | 비고 |
|------|---------|------|
| 결과 분석 깊이 (오답률, 과목 비교, 분포) | `exam/result/` | 소단기보다 훨씬 깊음 |

## 반영해야 할 항목 전체 목록

### ★★★ 즉시 반영 (이번 배포 전 필수)

---

#### 1. 메인 상단 "현재 운영 단계" 배지 + 대표 CTA 버튼

**두 분석 모두 최우선으로 꼽은 항목.**

**문제**: 사용자가 메인에 들어와도 "지금 뭘 해야 하는지" 바로 안 보임.

**구현 방향**:
- 관리자가 현재 단계를 설정 (`site.operationStage`: `pre` / `open` / `scoring` / `cutPublished`)
- 단계에 따라 메인 히어로 섹션에 배지 + 대표 버튼 1개 노출

```
[사전등록 진행 중]  →  [응시표 미리 등록하기] 버튼
[답안 입력 오픈]    →  [지금 바로 입력하기] 버튼 (강조)
[가답안 채점 중]    →  [내 성적 확인하기] 버튼
[1차 컷 공개]       →  [합격예측 보기] 버튼
```

**관련 파일**: `src/app/page.tsx`, `src/components/landing/HeroFallback.tsx`, `src/lib/site-settings.constants.ts`

---

#### 2. 비로그인 사용자에게 메인 개요 탭 공개

**현재 문제**: 비로그인 시 `ExamFunctionArea` 전체가 흐려져서 서비스 신뢰감이 전달 안 됨.

**구현 방향**:
- "합격서비스 메인" 탭(지역별 통계 카드)은 비로그인에도 공개
- 개인화 정보(내 점수, 내 순위)가 포함된 탭만 로그인 필요
- 탭 잠금 메시지를 "로그인 후 이용 가능" 으로 구분 표시

**관련 파일**: `src/components/landing/ExamFunctionArea.tsx`

---

#### 3. 예측 페이지 상단 운영 문장형 신뢰도 박스

**현재 문제**: 데이터는 풍부하지만 "이 데이터를 믿어도 되나?"라는 불안이 흡수되지 않음.

**구현 방향**:
- 합격예측/성적분석 탭 최상단에 운영 상태 안내 박스 추가
- 관리자가 설정한 운영 단계에 따라 자동으로 문구 변경

```
예시 문구:
─────────────────────────────────────────────────────────
⚠️ 현재 가답안 기준 예측값입니다. 정답 확정 전이므로 점수가 변동될 수 있습니다.
   정답 변경 시 자동 재채점되며 알림으로 안내드립니다.
─────────────────────────────────────────────────────────
✅ 정답이 확정되었습니다. 현재 점수는 최종 채점 기준입니다.
─────────────────────────────────────────────────────────
```

**관련 파일**: `src/app/exam/prediction/page.tsx`, `src/app/exam/result/page.tsx`

### ★★☆ 이번 시즌 내 반영 (시험 전후 반영 권장)

---

#### 4. FAQ를 단계별 운영형으로 재편

**현재 문제**: 정적 FAQ 목록 구조. "지금 필요한 질문"이 무엇인지 모름.

**구현 방향**:
- FAQ를 카테고리별로 분류 표시
  - 시험 전 / 답안 입력 / 점수·예측 / 정답 변경 / 합격컷 공개
- 각 섹션에 관련 기능 바로가기 버튼 추가
- 현재 운영 단계에서 중요한 카테고리를 자동 강조

**관련 파일**: `src/app/exam/faq/page.tsx`

---

#### 5. 알림 체계 확장 (운영 이벤트 중심)

**현재**: 재채점 알림만 존재 (NotificationBell).
**08_경쟁력강화 문서의 F-01과 연계.**

**확장 대상**:
- 답안 입력 오픈 알림
- 정답 변경 + 내 점수 영향 알림 (이미 기반 구현됨)
- 1차 컷 공개 알림
- 최종합격예측 오픈 알림
- 내 지역 표본 급증 알림

**관련 파일**: `src/components/layout/NotificationBell.tsx`, `src/app/api/admin/` 알림 관련 API

---

#### 6. 전체 지역 테이블 뷰 추가 (지역 버튼 없이 한눈에 비교)

**현재 문제**: 지역 버튼을 하나씩 클릭해야 해서 지역 비교 불편.

**구현 방향**:
- "합격서비스 메인" 탭 하단에 전체 지역 요약 테이블 추가 (접기/펼치기 가능)
- 컬럼: 지역 | 직렬 | 선발 | 참여인원 | 1배수컷 | 합격확실권 | 평균점수

**관련 파일**: `src/components/landing/ExamMainOverviewPanel.tsx`

---

#### 7. 단계별 합격컷 발표 시스템 (1차→최종 컷 공개 탭)

**08_경쟁력강화 문서의 F-02와 동일.**
소단기의 "3차컷 오픈" 탭에 해당.

**구현 방향**:
- 관리자가 단계별 확정 컷을 입력 (1차 예측 → 정답확정 후 재채점 → 경찰청 공식 발표)
- 공개 시점에 별도 탭 또는 섹션으로 표시
- 예측값 vs 실제 발표값 비교 표시 → 서비스 신뢰도 향상

---

#### 8. 합격예측 결과 페이지 단계형 타임라인 추가

**구현 방향**:
```
1. 사전등록  →  2. 답안 입력  →  3. 가채점 분석  →  4. 정답 확정  →  5. 1차 컷 공개  →  6. 최종 예측
    ✅              ✅               현재 단계              예정              예정             예정
```
- 현재 단계가 하이라이트됨
- 다음 단계 예정 날짜 표시

**관련 파일**: `src/app/exam/prediction/page.tsx` 또는 공통 컴포넌트
